## 改写策略

### 必须做的
1. **删除 AI 套话**：去掉"值得注意的是"、"需要指出的是"、"总的来说"、"综上所述"、"众所周知"、"不言而喻"等华而不实的过渡语。
2. **打破排比结构**：AI 倾向于写"首先…其次…最后…"或整齐的并列句，要把它们拆成长短不一的自然段落。
3. **加入设计权衡**：在关键设计点补充"之所以选这个方案是因为…"或"这里没有用XX是考虑到…"的理由说明。
4. **补充具体参数**：加入版本号（如 Python 3.10.12）、配置值（如超时 15s、分页 20 条）、具体端口号等工程细节。
5. **口语化转换**：用"这个模块"替代"该模块"，用"做到"替代"实现"，用"选了"替代"采用了"。
6. **句式变换**：主动句↔被动句混用，长句拆短句，短句合并，避免每段都是相同的"主语+谓语+宾语"节奏。

### 禁止做的
1. 不能改变技术内容的准确性。
2. 不能删除表格、列表等结构化内容——保持原样输出。
3. 不能大幅增删字数，改写后字数与原文偏差不超过 15%。
4. 不能添加新的技术细节——只改表述方式。
5. 不能输出任何解释说明，只输出改写后的完整章节文本。
