# 软著编写 Agent LLM 化改造需求

## 目标
在现有本地 MVP 基础上，将目录、说明书正文和源代码生成升级为大模型 API 驱动，并在关键阶段增加人工审查和人工修改能力。

## 需求
- 说明书目录生成应优先调用大模型 API，输入包含标题分析、目标字数、模块候选和软著标准目录约束。
- 说明书正文应逐章调用大模型 API，输入包含当前章节、目标字数、标题分析、目录上下文、前文摘要和模块元数据。
- 源代码生成必须在说明书生成完成后执行，按模块读取说明书上下文，要求大模型返回具体文件路径和文件内容。
- 运行过程增加人工审查点：目录审查、说明书章节审查、源代码审查。
- 人工审查既支持交互式暂停，也支持把可修改草稿写入 `review/` 目录后继续读取。
- 保留无 API Key 时的离线 fallback，保证测试和本地演示可运行；同时提供 `--llm-required` 强制要求 API。

## 验收标准
- 新增 LLM 客户端抽象，可通过 OpenAI-compatible Chat Completions 接口调用。
- 单元测试可用 fake LLM 验证目录、说明书、代码生成确实走 LLM 路径。
- CLI 暴露 LLM 配置和人工审查参数。
- 原有离线测试继续通过。
- 生成报告能说明 LLM/fallback 使用情况和审查目录位置。

## 非目标
- 不绑定某一个云厂商模型作为唯一选择。
- 不在测试中调用真实网络 API。
- 不承诺自动通过 AIGC 检测；风格改写仍定位为人工审校辅助。
