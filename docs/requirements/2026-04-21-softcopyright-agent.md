# 软著编写 Agent 需求说明

## 目标
基于 `软著Agent设计方案.md` 开发一个可本地运行的 Python Agent。用户输入软件著作权标题后，系统自动完成标题分析、目录生成、说明书撰写、源代码素材生成、文本风格改写和输出整理。

## 交付物
- Python 包 `softcopyright_agent`，包含六阶段流水线。
- 命令行入口，可通过 `python -m softcopyright_agent "标题"` 运行。
- Prompt 模板和代码结构模板，便于后续替换为真实大模型调用。
- 单元测试，覆盖标题分析、目录生成、文档生成、代码生成、输出格式化和端到端流程。
- 输出物包含 Markdown 说明书、简易 DOCX、源代码目录、生成报告和机器可读元数据。

## 功能范围
- Phase 1 标题分析：提取技术栈、业务领域、功能模块、关键词。
- Phase 2 目录生成：按软著通用说明书结构生成章节，并保持模块与代码规划一致。
- Phase 3 说明书编写：按章节目标字数生成中文设计说明内容，默认目标 9000 字。
- Phase 4 代码生成：按模块生成 Python 项目素材，默认不少于 3000 行。
- Phase 5 风格改写：清理模板化套话，调整表达方式，输出改写后的说明书内容；不承诺绕过任何外部检测系统。
- Phase 6 整理输出：写入说明书、源代码、报告和统计数据。

## 约束
- 仅依赖 Python 标准库，降低 Windows 本地运行门槛。
- 输出内容需与标题分析结果和模块列表保持一致。
- 代码行数、文档字数必须可统计、可校验。
- 生成 DOCX 使用标准库构造最小 OpenXML 文档，不引入第三方包。
- 当前仓库不是 Git 仓库，无法履行增量提交要求；执行结果需在报告中说明。

## 验收标准
- `python -m unittest discover -s tests` 通过。
- CLI 可完成一次低阈值端到端生成。
- 输出报告包含文档字数、代码行数、模块映射和文件清单。
- 默认配置支持 8000-10000 字说明书目标和不少于 3000 行源代码目标。

## 非目标
- 不接入真实大模型 API。
- 不实现 Web UI。
- 不实现真实 AIGC 检测 API。
- 不保证生成代码可作为生产系统部署，只作为软著申请源代码素材。
